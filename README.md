# ЭкоДесант MAX

Сервис для координации экологических и социальных волонтёрских акций внутри мессенджера MAX. Проект разработан для хакатона «Идеи для MAX: хакатон студенческих проектов СКФУ», кейс 4.1 «ЭкоДесант: координация волонтёрских акций».

Решение объединяет чат-бота и мини-приложение: бот регистрирует пользователей, помогает волонтёрам и организаторам, а мини-приложение даёт удобный интерфейс для просмотра акций, записи, создания мероприятий и подтверждения участия.

## Проблема

Организаторы волонтёрских акций часто сталкиваются с ручным учётом участников, неравномерной явкой, сложностью планирования инвентаря и долгой подготовкой отчётности после мероприятия. Волонтёрам, в свою очередь, нужен быстрый способ найти подходящую акцию, записаться, подтвердить участие и увидеть свой вклад.

`ЭкоДесант MAX` закрывает эти сценарии в одном пространстве MAX: регистрация, поиск акций, запись, управление событиями, QR-подтверждение участия и история волонтёрской активности.

## Основные возможности

### Для волонтёра

- регистрация профиля через чат-бота;
- выбор города по геолокации или вручную;
- просмотр доступных мероприятий в мини-приложении;
- поиск мероприятий по названию, городу и категории;
- просмотр подробной информации: дата, место, инвентарь, длительность, сертификат, свободные места;
- запись на мероприятие;
- отмена записи до подтверждения участия;
- отдельный раздел «Мои записи»;
- история завершённых мероприятий и количество часов;
- QR-код волонтёра для подтверждения участия организатором;
- ИИ-помощник по вопросам участия в волонтёрских акциях.

### Для организатора

- регистрация и авторизация роли организатора;
- создание мероприятия;
- указание города, адреса, категории, даты, времени и количества волонтёров;
- настройка инвентаря;
- указание длительности мероприятия в часах;
- выбор типа сертификата: международный, всероссийский, региональный, университетский или без сертификата;
- просмотр списка своих актуальных мероприятий;
- просмотр записавшихся участников;
- отмена мероприятия;
- сканирование QR-кода волонтёра для подтверждения участия;
- автоматический перевод прошедших мероприятий в статус завершённых;
- формирование отчёта по завершённому мероприятию через ИИ-помощника.

## Быстрый запуск локально

### 1. Клонировать репозиторий

```bash
git clone https://github.com/PyxaLabavyxa/HAKATON_MaxBot_EcoDesant.git
cd HAKATON_MaxBot_EcoDesant
```

### 2. Создать виртуальное окружение

Windows:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. Настроить переменные окружения

Создайте файл `MaxBot/.env` на основе примера:

```bash
cp MaxBot/.env-example MaxBot/.env
```

Для Windows PowerShell:

```powershell
Copy-Item MaxBot\.env-example MaxBot\.env
```

Заполните обязательные значения:

```env
BOT_TOKEN='YOUR_MAX_BOT_TOKEN'
AUTHORIZATION_KEY='YOUR_GIGACHAT_AUTHORIZATION_KEY'
```

Если нужно изменить адрес или порт web-сервера, добавьте необязательные параметры:

```env
WEBAPP_HOST='0.0.0.0'
WEBAPP_PORT=8080
```

`BOT_TOKEN` выдаётся для чат-бота MAX. `AUTHORIZATION_KEY` используется для функций ИИ-помощника и генерации отчётов. Если `WEBAPP_HOST` и `WEBAPP_PORT` не указаны, приложение автоматически использует `0.0.0.0:8080`.

### 4. Подготовить базу данных

Проект использует SQLite-файл:

```text
MaxBot/database/maxbot.db
```

### 5. Запустить бота и web-сервер

```bash
cd MaxBot
python main.py
```

После запуска:

- бот работает через polling;
- мини-приложение и API доступны на `http://localhost:8080/`;
- статические файлы отдаются тем же aiohttp-сервером.

## API мини-приложения

| Метод | URL | Назначение |
| --- | --- | --- |
| `GET` | `/api/user?max_user_id=...` | получить пользователя, роль и данные для интерфейса |
| `GET` | `/api/events?city=...&max_user_id=...` | получить список мероприятий |
| `GET` | `/api/history?max_user_id=...` | получить историю волонтёра |
| `POST` | `/api/join` | записаться на мероприятие |
| `POST` | `/api/cancel-registration` | отменить запись |
| `POST` | `/api/events` | создать мероприятие |
| `GET` | `/api/events/{event_id}/volunteers` | получить список участников мероприятия |
| `POST` | `/api/events/{event_id}/cancel` | отменить мероприятие |
| `POST` | `/api/events/{event_id}/confirm` | подтвердить участие волонтёра |

## Подключение мини-приложения к MAX

Для работы внутри MAX мини-приложение должно быть доступно по HTTPS.

1. Разверните проект на сервере.
2. Настройте домен или поддомен с HTTPS.
3. Проксируйте внешний HTTPS-трафик на web-сервер приложения, например на порт `8080`.
4. В кабинете MAX для партнёров укажите URL мини-приложения.
5. Откройте мини-приложение через кнопку в боте.

Текущая логика мини-приложения получает идентификатор пользователя из MAX WebApp Bridge. Если пользователь не зарегистрирован в боте, мини-приложение показывает экран с просьбой сначала пройти регистрацию.

## Пример сценария демонстрации

1. Пользователь запускает бота командой `/start`.
2. Проходит регистрацию как волонтёр: город, ФИО, дата рождения, телефон.
3. Открывает мини-приложение из бота.
4. Выбирает мероприятие и записывается на него.
5. Организатор открывает свою панель, видит участника в списке записавшихся.
6. Волонтёр показывает QR-код из бота.
7. Организатор сканирует QR-код в мини-приложении и подтверждает участие.
8. У волонтёра появляется запись в истории, а организатор может сформировать отчёт.
